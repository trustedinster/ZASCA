## 版本命名规范

### 语义化版本控制
我们遵循 [语义化版本控制 2.0.0](https://semver.org/lang/zh-CN/) 规范：

- **主版本号**：当你做了不兼容的 API 修改
- **次版本号**：当你做了向下兼容的功能性新增
- **修订号**：当你做了向下兼容的问题修正

### 版本标签格式
```
v主版本号.次版本号.修订号[-预发布标识][+构建元数据]
```

示例：
- `v3.0.0` - 正式发布版本
- `v3.1.0-beta.1` - Beta 预发布版本
- `v3.1.0+build.123` - 包含构建元数据

## 发布周期

### 版本发布时间表
- **主版本**：每 6-12 个月发布一次重大更新
- **次版本**：每 2-3 个月发布一次功能更新
- **修订版本**：每月发布安全补丁和 bug 修复

### 发布流程
1. **开发阶段**：功能开发和单元测试
2. **测试阶段**：集成测试和用户验收测试
3. **预发布**：Beta 版本公开测试
4. **正式发布**：生产环境部署
